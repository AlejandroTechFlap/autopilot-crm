# Spanish UI Convention & Glossary [IMPLEMENTED]

## Rule

The Autopilot CRM UI is **Spanish-only**. Every user-visible string (labels,
buttons, placeholders, toasts, tooltips, modal titles, table headers, empty
states, error messages surfaced via `toast.error`, etc.) must be in Spanish.

This is the documented exception to the global "English only" rule defined in
`~/.claude/CLAUDE.md` §0: code, comments, commit messages, docs and memory stay
in English — only **user-facing UI strings** are localized.

## Where labels live

- **Enum-driven labels** (lifecycle, source, priority, role, activity type,
  category): `src/lib/constants.ts`. Always render through these maps, never
  hardcode the enum value in JSX.
- **Date / relative time strings**: `src/lib/formatting.ts`
  (`formatRelativeTime`).
- **Top-bar route titles**: `src/components/layout/top-bar.tsx → ROUTE_TITLES`.
- **Sidebar nav labels**: `src/components/layout/sidebar.tsx → NAV_ITEMS`.
- **Per-page strings**: inline in the component file.

## Process for new strings

1. Check the glossary above. If the term exists, reuse the exact translation.
2. If it's missing, add a row to the glossary in the same PR that introduces it.
3. Never hardcode an English string "temporarily" — Spanish from day one.
